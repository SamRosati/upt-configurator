 
UPT Configurator – High-level Project Specification (Guideline) 
Project Brief Summary 
This project aims to design and deploy a web-based 3D product configurator for the UPT e-commerce 
portal, enabling customers and partners to visually build, customize, and configure modular vehicles 
online and proceed directly to checkout. 
UPT (Utility Personal Transporter) is ENVO’s modular vehicle architecture built around a standardized 
main frame (“main block”) that supports multiple vehicle types, ranging from consumer mobility 
solutions to industrial and utility applications, by attaching interchangeable suspension systems, 
drivetrains, bodies, and accessories. UPT is designed to maximize modularity, scalability, and cross-
compatibility across products while reducing cost, complexity, and time to market. 
 
The configurator will function similarly to IKEA’s cabinet configurator, offering real-time 3D visualization 
with rotation, zoom, and step-by-step assembly of frames, modules, and accessories. It will act as a 
core marketing, sales, and operations enablement tool, reducing manual project discussions with DIY 
buyers and industrial partners, while streamlining the flow from configuration to POS, inventory 
management, and manufacturing. It will also serve as an educational and onboarding platform for 
dealers, upfitters, and assembly partners.

Initially, the configurator will support limited, sellable configurations across all single-frame UPT 
products, including Platform, Essential UPT (4×4 scooter), standard eATV, standard LSV, golf cart, 
cargo bike, industrial transporter, buggy, and compatible accessories, kits, and packages. New 
components and vehicle designs must be addable on an ongoing basis, including “coming soon” 
configurations to test market demand or signal future compatibility for existing customers. 
The system must support two user journeys on the same configurator engine: 
1.an advanced journey for industrial partners, manufacturers, research institutions, and DIY 
users, allowing them to build vehicles from a bare main frame and add components layer by 
layer; and 
2.a simplified consumer journey starting from pre-configured vehicles with guided customization 
options. 
At any stage, the configurator must generate a unique, shareable link that reproduces the exact 
configuration for internal technical or commercial discussions, customer sharing, social media, and 
sales negotiations. 
The platform must be architected for phased expansion, including multiple frame sizes (e.g., extended 
length, double-wide) and additional UPT platform families. The solution must be built on reliable, open-
source, industry-standard technologies, avoiding vendor lock-in or ongoing licensing fees. Custom 
development is acceptable, provided the underlying technology is widely known, maintainable by third-
party experts, and secure for long-term use.

1. Overview 
● Objective: Build a 3D web configurator for UPT modular vehicles on a new Shopify storefront, 
enabling both B2C preset customization and advanced build-from-frame assembly. 
● Phase 1 scope: Advanced journey (Build‑from‑Frame) plus consumer presets, with live pricing, 
add-to-cart, save/share, multilingual/multicurrency, quotes/PDF, rules with block/suggest, 
“coming soon” flows, analytics/GA4, basic admin for updates. 
● Attachments to govern details (provided separately): Assembly logic and matrix, Website 
wireframe/UX/UI and use journeys, CAD/asset samples. 
 
2. Target audiences and journeys 
● Audiences: 
○ Consumers: Quick start from presets (e.g., Golfcart, eATV, LSV) with accessories. 
○ DIY/industrial/educational: Advanced build-from-frame flow with multi-stage assembly

○ Dealers: Use the same configurator with dealer pricing via Shopify (dealer specifics

● Journeys in Phase 1: 
○ Consumer presets: Start from a preconfigured vehicle and customize options.

○ The configurator will use a Stateless Recipe Code to represent any build. The code

an integrity check, and allows the app to fully reconstruct the 3D build on load, 
re‑validate rules, and recalculate pricing from live Shopify data. If a code references 
deprecated parts, the app blocks the load and suggests valid alternatives (or opens in 
an ‘archived’ view per policy). For ‘Save,’ we store only the Recipe Code: guests must

○ Desktop latest Chrome/Edge/Safari/Firefox

○ Initial 3D payload 8–12 MB; lazy-load additional assemblies 
○ Time-to-first-interaction ≤ 4s on mid-range mobile over 4G/LTE

○ Textures capped at 2K; use atlases; at least 2 LODs per large assembly 
● Fallbacks: 
○ Auto-switch to lightweight model on low-end devices; last resort static 360 imagery 
prompt

6. Platform and integrations (to be decided in Phase 0) 
● Platform approach: Open-source/self-hosted vs vendor platform vs hybrid. Decision to be made

7. Configuration logic and rules model 
● Rule types: 
○ Requires: A requires B 
○ Excludes: A blocks B 
○ One-of group: choose exactly one from group 
○ Capacity/fit checks: wheelbase/frame/body fit, powertrain compatibility, tire clearance, 
electrical limits 
● Behavior: On incompatible selection →  block with explanation and suggest valid alternatives 
(R2). 
● Authoring/updates: Rules provided via CSV matrix and/or lightweight admin UI; vendor to define 
a simple JSON/CSV schema for import (IDs, groups, requires/excludes, conditions). 
● Versioning: To be confirmed (whether old shared builds reopen with archived parts or map to 
nearest current equivalents).
